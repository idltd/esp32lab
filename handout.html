HC-SR04 — Ultrasonic Distance (5V version)
SensorESP32
VCC→5V (USB/VIN pin)
GND→GND
TRIG→GPIO 4 (direct)
ECHO→GPIO 3 via divider ↓
ECHO ──┬── GPIO 3 │ 10kΩ │ ┴ (10k+20k divider) 20kΩ gives 3.3V from 5V │ GND
5V ECHO pin will damage the ESP32 if connected directly. Voltage divider: 10kΩ from ECHO to GPIO 3, then 20kΩ from GPIO 3 to GND. TRIG is fine direct — it is an input on the sensor and accepts 3.3V logic.
DHT11 — Temperature & Humidity
SensorESP32
VCC→3.3V
GND→GND
DATA→GPIO 4
✓ 3.3V safe — connect directly.
Temperature ±2°C  |  Humidity ±5% RH  |  1 reading/sec max.
Uses a single-wire serial protocol — all data travels on one wire using its own timed bit-banging protocol (not I²C or UART). Breakout modules include a built-in pull-up resistor; bare sensors need 10kΩ DATA→3.3V.
PIR — Motion Sensor (as Digital Input)
SensorESP32
VCC→5V (most modules)
GND→GND
OUT→GPIO 4
OUT goes HIGH when motion detected, LOW after the delay period. Select Digital Input in the sensor dropdown.
Jumper H Retrigger — timer resets while motion continues
Jumper L Single-shot — ignores motion until delay expires
Sensitivity trim Turn clockwise for longer detection range
Delay trim How long OUT stays HIGH after motion stops
PWM Output (LED dimming / buzzer)
SensorESP32
Anode (+)→GPIO 4 via 330Ω
Cathode (−)→GND
GPIO 4 → 330Ω resistor → LED (+) → LED (−) → GND.
Duty cycle slider: 0 = off  |  128 = 50%  |  255 = full brightness.
Can also drive a buzzer or relay module directly on GPIO 4.
Digital HIGH / LOW
Input (button / switch)
Signal→GPIO 4
Other leg→GND
Use INPUT_PULLUP mode. Pin reads LOW when button pressed, HIGH when released.
Output (LED / relay)
GPIO 4→330Ω → LED → GND
Write HIGH to turn on, LOW to turn off. Works with relay modules, buzzers, indicator LEDs.
